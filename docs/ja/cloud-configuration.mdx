## XC (UI)

[L3/L4 Routed DDoS Mitigation](https://docs.cloud.f5.com/docs-v2/ddos-and-transit-services/how-tos/network-firewall/l3l4-ddos-mitigation) ガイドに基づき、Webインターフェースを使用して **Cloud** 側を構成します。

### DDoS ワークスペースの有効化

トンネルと BGP を構成する前に：

1. Cloud チーム（`sales@example.com` またはアカウントチーム経由）に連絡して以下を依頼します：
    - テナントの **Routed DDoS Mitigation** ワークスペースを有効にする。
    - 以下を提供する：
        - **パブリック IP ネットブロック**（IPv4 の場合 /24 以上、IPv6 の場合 /48 以上）と所有権の証明（または LOA）。
        - **ASN**（ARIN または同等のレジストリによって発行されたもの）。
        - **IRR** 登録（RIPE、ARIN、APNIC など）および RPKI リポジトリの **ROA**。
        - 希望する**クリーントラフィックの返送方法**（GRE トンネル、Layer 2、Equinix 経由のプライベートピアリングなど）。
        - 保護対象のデータセンターの場所とルーター。
        - ルートアナウンスメントに必要な **AS-Path プリペンド**。
        - **BGP コミュニティ**、ルートプリファレンス、または **AS-SET**

1. SOC が以下をプロビジョニングします：
    - トンネル
    - ASN
    - 保護対象プレフィックス
    - ルート広告オプション
    - 必要に応じた Fast ACL / ファイアウォールポリシー

### トンネル

1. [Cloud Console](https://docs.cloud.f5.com/docs-v2/ddos-and-transit-services/how-tos/network-firewall/l3l4-ddos-mitigation) にログインし、サービスセレクターから **Routed DDoS** を選択します。
1. **Manage > Tunnels > Add Tunnel** に移動し、以下を構成します：
    - **Location Name** と **Availability Zone**（デフォルトは Zone 1）。
    - **Bandwidth Max in MB**。
    - **Tunnel Type**：
        - **GRE Over IPv4**：IPv4 外部トンネル用。
        - **GRE Over IPv6**：IPv6 外部トンネル用。
        - **IP Over IP**：IPv4-in-IPv4 カプセル化用。
        - **IPv6 Over IPv6**：IPv6-in-IPv6 カプセル化用。
    - **Customer Endpoint IP**：BIG-IP の外部アドレス（外部セルフ IP、トンネルがパブリックインターネットを通過する場合はパブリックにルーティング可能である必要があります）。
    - オプション：IPv4/IPv6 インターコネクト、フラグメンテーション、キープアライブ（すべてデフォルトで無効）。
1. **Tunnel BGP Information** の下で：
    - **ASN object**（お客様の ASN）を選択します。
    - **Customer Peer Secret Override** を設定します：Use Default Secret（デフォルト）、BGP Password Override（ブラインドまたはクリアテキスト）、または No Secret。
    - デフォルトと異なる場合は **Holddown Timer** の値を秒単位で設定します。

SOC がこれらのトンネルオブジェクトを事前に作成する場合があります。BIG-IP 上で**エンドポイント IP** と BGP 設定を一致させるだけです。

### ASN とルート

- **ASN**：-> **Manage > ASNs > Add ASN**。
    - ASN を入力し、BGP が有効になっていることを確認します。

- **プレフィックス**：-> **Manage > Prefixes > Add Prefix**。
    - 各 IP プレフィックスを入力し、ASN に関連付けます。

- **ルート広告** -> **Manage > Route Advertisement > Add Route Advertisement**。
    - プレフィックスを入力し、**Active** または **Not Advertised** を選択し、オプションで有効期限を設定します。

これらのオブジェクトは、サービスがアクティブな場合にグローバルネットワーク経由でアナウンスされるプレフィックスを制御します。

### ネットワーク制限

**Firewall Rules**、**Deny List Rules**、および **Fast ACLs for Internet VIPs** により、以下が可能です：

- 特定のトラフィックをブロックまたは許可する。
- 不正なソースをレート制限する。
- 純粋なボリュメトリックスクラビング以上の追加 DDoS 保護を適用する。
